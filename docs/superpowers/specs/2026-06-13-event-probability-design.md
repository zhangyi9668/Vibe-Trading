# 事件概率模块设计

## 目标

在 Vibe Trading Web UI 左侧边栏新增“事件概率”模块，整合 Polymarket 与 Kalshi 的公开预测市场数据，作为宏观情绪和风险温度计。

模块不提供交易操作，也不把事件概率解释为买卖信号。

## 页面入口

- 左侧边栏新增“事件概率”。
- 前端路由为 `/event-probability`。
- 页面采用模块总览型布局。
- 页面不展示第三方项目来源说明。

## 页面结构

顶部状态区显示：

- 当前快照时间。
- 当前事件数量。
- Polymarket 与 Kalshi 数据源状态。
- 翻译缓存命中率。
- 当前刷新状态和进度。

顶部操作区提供：

- 快速刷新。
- 完整刷新。

筛选区提供：

- 宏观模块。
- 数据来源。
- 关键词。
- 概率变化范围。

主体按以下模块分区展示：

- 货币政策。
- 宏观经济。
- 地缘政治。
- 政治选举。
- 股指大宗。
- AI 科技。
- 加密。
- 体育。
- 娱乐。
- 其他。

每条事件显示：

- 中文标题。
- 英文原文。
- 当前概率。
- 24 小时概率变化。
- 24 小时成交量。
- 数据来源。
- 到期时间。

Polymarket 事件可以展开查看概率趋势图。Kalshi 第一版只展示当前概率，不实现历史趋势。

页面底部显示“风险温度计，不构成交易信号”的提示。

## 后端架构

所有第三方请求由 Python 后端执行，前端只访问 Vibe Trading 自身 API。

后端包含以下职责：

1. Polymarket 数据抓取和字段标准化。
2. Kalshi 数据抓取和字段标准化。
3. 统一分类、去重和模块限流。
4. 快照读取与原子写入。
5. 刷新任务状态管理。
6. 标题翻译队列与永久缓存。

标准化事件至少包含：

- `question`
- `question_zh`
- `topic`
- `prob_yes`
- `change_24h`
- `volume_24h`
- `liquidity`
- `source`
- `end_date`
- `slug`
- `token_id_yes`
- `pick_label`

## 刷新策略

### 页面加载

页面优先读取本地快照，不等待第三方 API。存在有效快照时应立即展示。

### 快速刷新

快速刷新用于日常更新：

- Polymarket 按 24 小时成交量获取热门开放市场。
- Kalshi 并发更新已知重点系列，包括美联储、CPI、就业、经济增长和主要商品等宏观系列。
- 快速刷新不扫描 Kalshi 全部开放事件。
- 目标完成时间为 5 至 20 秒，实际时间取决于第三方 API。

### 完整刷新

完整刷新用于发现新事件和新系列：

- 后台分页扫描 Kalshi 开放事件。
- 前端继续展示旧快照。
- 前端轮询刷新状态和进度。
- 完成后原子替换快照。
- 新发现的高价值系列进入后续快速刷新清单。

完整刷新可以由用户手动触发，后续可增加定时任务，但不属于第一版范围。

### 并发约束

- 快速刷新和完整刷新互斥。
- 同一时间只允许一个刷新任务。
- 重复请求返回当前任务状态，不创建重复任务。

## 翻译策略

标题翻译通过当前 Vibe Trading LLM 配置执行。

- 只翻译首次出现的新英文标题。
- 翻译结果按英文原文永久缓存。
- 已缓存标题不再消耗 LLM Token。
- 每批最多翻译 4 条。
- 批次间延迟约 1 秒。
- 快速刷新每次最多翻译 30 条新标题。
- 完整刷新每次最多翻译 100 条新标题。
- 超出额度的标题保留英文，后续刷新逐批补齐。
- 翻译失败不影响数据快照和页面展示。

刷新状态中记录：

- 本次新翻译数量。
- 翻译缓存命中数量。
- 等待翻译数量。

## 缓存与失败处理

- 快照写入采用临时文件加原子替换。
- 任一数据源返回空结果或请求失败时，不覆盖该数据源的上一份有效数据。
- 单一数据源失败时，另一数据源仍可更新。
- 完整刷新失败时保留旧快照，并在页面显示错误摘要。
- 第三方字段解析失败时跳过单条异常数据，不中断整批刷新。
- 翻译服务不可用时显示英文原文。

## API 设计

第一版提供以下后端接口：

- `GET /event-probability/overview`
  - 返回当前快照、数据源状态和刷新状态。
- `POST /event-probability/refresh/quick`
  - 启动快速刷新。
- `POST /event-probability/refresh/full`
  - 启动完整刷新。
- `GET /event-probability/refresh/status`
  - 返回任务类型、阶段、进度和错误摘要。
- `GET /event-probability/history/{token_id}`
  - 返回 Polymarket 概率趋势。

刷新接口只负责启动任务并快速返回，不保持长连接等待整个任务完成。

## 测试范围

后端测试覆盖：

- Polymarket JSON 字符串字段解析。
- Kalshi `*_dollars` 字段解析。
- Kalshi 多腿事件的代表概率选择。
- 事件分类、去重和模块限流。
- 快照原子写入。
- 空结果不覆盖旧快照。
- 单源失败回退。
- 刷新任务互斥。
- 翻译缓存和 30/100 条额度。

前端测试覆盖：

- 侧边栏入口和路由。
- 快照加载、筛选和模块展示。
- 快速刷新与完整刷新状态。
- 旧快照继续展示。
- 英文回退和错误提示。

## 许可证处理

页面不展示 globalpercent 来源说明。若实现中移植其 Apache 2.0 代码，则在源码或许可证材料中保留必要声明，不在用户界面展示。

## 第一版范围外

- 预测市场交易或账户连接。
- Kalshi 历史趋势图。
- 自动交易信号。
- 基于 LLM 的事件解读或投资建议。
- 自动定时完整刷新。
