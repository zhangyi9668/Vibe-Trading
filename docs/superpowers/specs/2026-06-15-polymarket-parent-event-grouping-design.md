# Polymarket 父事件聚合与多结果趋势图设计

## 目标

事件概率页面不再把同一 Polymarket 父事件下的不同结果拆成多张独立卡片。页面以父事件为一个主题，展示成交额最高的前 5 个结果，并在同一张趋势图中绘制这些结果的概率变化。

本次仅调整 Polymarket。Kalshi 保持现有数据结构和展示方式。

## 数据模型

为 Polymarket 市场保留父事件信息：

- `event_id`：Polymarket 父事件的稳定标识。
- `event_title`：父事件标题。
- `event_slug`：父事件链接标识。
- 子市场继续保留标题、概率、24 小时变化、成交额和 Yes Token ID。

后端概览模型增加父事件分组结构。每个分组包含：

- 父事件标题及中文翻译。
- 主题分类、到期时间和来源信息。
- 子结果列表。
- 子结果成交额之和，作为父事件总成交额。

若市场数据没有可靠的父事件 ID，则不使用标题模糊匹配，继续按现有单条市场展示，避免误合并。

## 分组与排序

Polymarket 数据获取完成后执行以下处理：

1. 按真实 `event_id` 聚合同一父事件的子市场。
2. 父事件内部按子市场 `volume_24h` 从高到低排序。
3. 每个父事件只保留前 5 个子结果。
4. 父事件总成交额使用全部子市场成交额之和，避免截断结果影响主题排名。
5. 页面每个分类仍只保留总成交额最高的 8 个主题。

没有父事件信息的市场作为单结果主题参与相同的分类和 Top 8 排序。

## 历史数据接口

新增支持多个 Token ID 的历史查询接口。接口接收最多 5 个 Token ID，并行请求 Polymarket 历史价格：

- 每条曲线返回结果名称、Token ID 和时间点数组。
- 单条曲线请求失败时保留其他成功曲线。
- 所有曲线失败时返回明确错误。
- Token ID 继续执行格式校验，并限制数量，避免无限并发。

原有单 Token 历史接口暂时保留，避免影响其他调用。

## 前端展示

Polymarket 父事件使用一张主题卡片：

- 卡片标题显示父事件中文标题，英文标题作为辅助信息。
- 结果区域显示成交额最高的前 5 个结果。
- 每个结果显示名称、当前概率、24 小时变化和成交额。
- 结果按成交额从高到低排列。
- 卡片底部直接显示共用趋势图，不需要展开操作。

趋势图：

- 最多绘制 5 条不同颜色的折线。
- 图例显示结果名称和当前概率。
- Y 轴固定为 0% 到 100%。
- 各曲线使用统一时间轴；缺失时间点保持空缺，不伪造概率。
- 保留进入视口后再加载历史数据的机制，控制初次页面请求数量。

Kalshi 卡片继续使用当前单结果布局。

## 翻译

翻译对象调整为父事件标题和最终展示的前 5 个结果名称：

- 使用现有 Codex Auth。
- 快速刷新最多新增 30 条翻译。
- 完整刷新最多新增 100 条翻译。
- 已缓存翻译不重复调用模型。
- 翻译失败时保留英文，不阻断数据刷新。

翻译调用需要使用 ChatGPT Codex Auth 支持的兼容模型，不修改用户配置的主对话模型。

## 错误处理

- 父事件字段缺失：退回单结果主题。
- 部分历史曲线失败：展示成功曲线并提示部分数据不可用。
- 全部历史曲线失败：显示趋势加载失败状态。
- 中文翻译失败：显示英文原文。
- Polymarket 数据结构异常：跳过异常市场，不影响其他市场刷新。

## 测试

后端测试覆盖：

- 按真实父事件 ID 分组。
- 不按相似标题误合并。
- 子结果按成交额选前 5 个。
- 父事件总成交额使用全部子市场求和。
- 分类 Top 8 使用父事件总成交额排序。
- 多 Token 历史并行查询、部分失败和数量限制。
- 翻译只处理最终展示标题并使用兼容 Codex 模型。

前端测试覆盖：

- 同一父事件只出现一张卡片。
- 卡片只显示前 5 个结果。
- 结果顺序按成交额排列。
- 多条历史曲线在同一图表中生成。
- Kalshi 继续使用原展示方式。
- 缺少父事件信息的 Polymarket 市场仍可显示。

## 不在本次范围

- Kalshi 父事件聚合。
- 使用标题相似度或 LLM 推断父事件关系。
- 展示超过 5 个子结果的展开功能。
- 改造 Polymarket 之外的数据源。
